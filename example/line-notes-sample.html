Line Notes Sample

This file shows the Markdown input, preview, and HTML output contract used by line-notes and the short alias notes. The preview column also demonstrates a few suggested CSS presentation patterns.

Multiple short inline notes

This uses 10 varied code lines plus several short notes. Most notes use the default width, and one line narrows locally to {width="8rem"}.

Long code lines (suggested fallback)

This block uses 10 long code lines plus several notes. The preview intentionally forces the below-block fallback so the long code stays legible; in that mode the notes use the same compact label-chip treatment as the non-anchor fallback, without full-width comment boxes. The emitted HTML still keeps the anchor metadata.

Markdown

```txt
const requestKey = buildCompositeKey(accountId, region, featureFlags, retryBudget)
const preparedInput = mergePrimaryAndShadowInputs(primaryInput, shadowInput, defaultsMap)
const snapshot = checkpoint_store.begin_snapshot(requestKey, preparedInput)
const normalizedBundle = normalize_bundle_for_transport(preparedInput, schema_version, locale_hint)
const dispatchResult = dispatch_bundle_through_pipeline(normalizedBundle, retry_plan_compact, observer_hooks)
const cacheToken = write_expanded_snapshot_cache(snapshot, normalizedBundle, dispatchResult.timeline)
publish_bundle_summary(accountId, requestKey, cacheToken, dispatchResult.timeline, observer_hooks)
release_snapshot(snapshot, cacheToken, dispatchResult.timeline, metrics_collector)
return finalize_published_bundle(dispatchResult, cacheToken, metrics_collector, observer_hooks)
cleanup_observer_handles(observer_hooks, cacheToken)
```
```line-notes
2: merge incoming inputs before snapshot
5: main latency hotspot
7: publish visible summary
9: final response assembly
```

Long note width tuning
